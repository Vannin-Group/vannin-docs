Greencube by Vannin Group

API overview

Greencube exposes a FHIR-based integration surface for customers and approved partners. This page describes how access is structured; full specifications are provided with credentials.

How access works

  • Scoped credentials. API clients are registered per organisation and per environment, with scopes limited to the resources each integration needs.
  • Sandbox first. Integrations are built and conformance-tested against a sandbox environment with synthetic data before any production access.
  • FHIR resources. Patient, Encounter, Observation, MedicationRequest, DiagnosticReport, and Appointment are the core exchange resources.
  • Event delivery. Subscriptions and HL7 v2 feeds are available for systems that need push delivery rather than polling.

Who integrates

Hospital IT teams connecting existing systems, national programmes connecting reporting infrastructure, and product partners in laboratory, imaging, and pharmacy. All integrations run through the same governance: registration, sandbox conformance, and production approval.
